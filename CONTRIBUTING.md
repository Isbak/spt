# Contributing

## Community standards

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
Report unacceptable behavior through the repository owner's preferred private
contact channel. Report security-sensitive concerns using [SECURITY.md](SECURITY.md).

## Rules

- Use pull requests.
- Do not commit directly to `main`.
- Run `make verify` before pushing.
- Update ADRs for architectural decisions.
- Update C4 docs for structural changes.
- Add tests for new functionality.
